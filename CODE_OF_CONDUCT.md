# Code of Conduct

OMNIX is a source-available commercial project with public issues and review
surfaces. Professional, good-faith collaboration is expected from maintainers,
contributors, evaluators, and reviewers.

## Expected Behavior

- Be respectful and direct.
- Keep technical criticism specific and actionable.
- Assume public conversations may be read by candidates, customers, auditors,
  and security reviewers.
- Protect private information, credentials, customer code, and vulnerability
  details.
- Respect the source-available license terms in `LICENSE.md`.

## Unacceptable Behavior

- Harassment, insults, personal attacks, or discriminatory language.
- Publishing private security details, credentials, customer data, or proprietary
  third-party code.
- Repeated off-topic comments, spam, or bad-faith issue activity.
- Pressuring maintainers to accept license, roadmap, or support obligations that
  have not been agreed in writing.

## Reporting

For conduct concerns, open a GitHub issue with only the minimum public context
needed to request a private channel. For security-sensitive concerns, follow
`SECURITY.md` instead and do not disclose vulnerability details publicly.

Maintainers may edit, hide, lock, or remove comments and issues that violate
this code, and may block repeat offenders from project interaction surfaces.
