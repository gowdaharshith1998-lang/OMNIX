# Security Policy

Please report security issues privately. Do not open public issues for
vulnerabilities, exploit details, credentials, or customer data.

## Reporting

Open a private GitHub security advisory when available, or request a secure
reporting channel through a repository issue without disclosing details.

Include:

- affected component or path
- impact and exploitability
- reproduction steps or proof of concept
- relevant logs with secrets removed

## Scope

Security-sensitive areas include verification subprocess execution, signed
receipts, cloud authentication, tenant isolation, GitHub App webhooks,
deployment manifests, and air-gapped bundle handling.

OMNIX is source-visible commercial software. Security reports do not grant a
license to use OMNIX commercially.
